Project Description

AI trading interfaces can generate transactions, but they cannot guarantee that a model-generated action respects the user's limits. A stale quote, hallucinated route, or compromised frontend can turn a convenient natural-language request into an unsafe financial transaction.

Aegis is an AI-native intent engine for Sui. A user describes a goal such as "Swap 1 SUI to USDC with maximum 1% slippage." OpenAI converts the request into a strictly validated SUI/DBUSDC intent. Aegis then reads live DeepBook market data, deterministically scores price impact, liquidity depth, market spread, and freshness, and explains those immutable results in plain English.

Before signing, the user sees a human-readable preview generated from the same typed ExecutionPlan used to build the transaction. Aegis dry-runs the exact Programmable Transaction Block, requires explicit confirmation, and requires additional acknowledgement for warning-level risk.

Sui is the security layer, not a payment rail added afterward. A user-owned GuardianPolicy Move object enforces direction-specific spending ceilings, maximum slippage, one allowed DeepBook pool, expiration, ownership, and persistent revocation. The final wallet-signed PTB atomically asserts that policy, executes the real DeepBook swap, and mints an owned IntentReceipt while emitting an IntentExecuted event. If any step fails, the entire transaction reverts.

Aegis currently runs on Sui testnet with an Explorer-verifiable package, active policy, real DeepBook execution, receipt, event history, analytics, and a deterministic non-executable stress mode. Next steps include additional DeepBook markets, Walrus-backed Guardian snapshots, Seal-encrypted private preferences, independent risk attestations, an audit, and mainnet deployment.
