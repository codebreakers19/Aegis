How it works

From words to
atomic transactions

Every step is verifiable. Every risk is named. Every signature is informed.

🗣️
Natural language intent
Type what you want in plain English. No pool IDs, no slippage math, no token addresses.
🧠
AI intent parsing
OpenAI gpt-4o-mini extracts assets, amount, slippage, and risk tolerance into a validated schema. Ambiguous amounts are rejected.
🛡️
Guardian risk check
Live DeepBook order book, oracle timestamps, and liquidity depth are fetched. Four risk classes scored deterministically.
📋
Human-readable PTB preview
See every step of the transaction in plain language before signing. No surprises.
⚡
Atomic execution
One signature. One transaction. All steps succeed or the whole thing reverts — no partial state.

Built different

Sui-native, not just Sui-compatible

Every component is load-bearing. Remove the chain, the system breaks. That's the point.

⟡
PTBs as first-class output
The compiler chains transaction outputs directly — the swap result feeds the receipt mint with no intermediate wallet state. This atomicity is impossible on any other chain.
Programmable TX Blocks
🛡️
VM-enforced policy
GuardianPolicy is a Move object — the chain physically enforces max slippage, max amount, allowed pools, and expiry. Bypassing the UI doesn't bypass the rules.
Move Objects
📋
On-chain audit trail
Every execution mints an IntentReceipt — transferable, composable, readable by other protocols. Your trade history lives on-chain, not in a database.
IntentReceipt NFT
📊
Live DeepBook data
Guardian reads actual order book depth, mid-price, and recent trades — not simulated risk. Score changes if the market moves before you sign.
DeepBook SDK
🔒
Explicit confirmation gate
Warn state requires reading and acknowledging risks. Block state removes the execute button entirely. No dark patterns.
Sub-track 3 compliant
🧪
Stress mode for demos
Clearly labeled deterministic snapshots let you demonstrate warn and block states reliably without pretending the data is live. Honest by design.
